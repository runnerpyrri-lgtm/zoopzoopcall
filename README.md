# 줍줍콜

무순위 · 잔여세대 · 취소후재공급 청약(줍줍) 접수가 **열리고 닫히는 순간**을, 내 조건에 맞는 것만 폰으로 미리 울려주는 알림 서비스.

**바로 열기: https://runnerpyrri-lgtm.github.io/zoopzoopcall/**

폰에서 홈 화면에 추가하면 앱처럼 쓸 수 있습니다 (안드로이드 크롬: 메뉴 → 홈 화면에 추가 / 아이폰 사파리: 공유 → 홈 화면에 추가).

## 핵심 가치

1. **모아준다** — 전국 3종 줍줍 공고를 한 곳에, 지역·유형·접수여부로 필터.
2. **놓치지 않게** — 접수 시작/마감 카운트다운 + 알림 프리셋(1일 전·3시간 전·정각/1시간 전).
3. **챙겨준다** — 알림 켜기 한 번으로 시작·마감 알림 예약.

## v0.1.0 현황

- 웹앱(PWA) 우선 출시. 어떤 폰·모니터 크기에서도 동작하는 반응형 + 라이트/다크 지원.
- 알림은 **로컬(브라우저) 알림**: 앱이 열려 있거나 홈 화면 앱으로 실행 중일 때 울린다. 완전 종료 상태에서도 오는 서버 푸시는 v0.2.0.
- 데이터는 **청약홈 실공고**만 표시한다. 실데이터 연결 전에는 임의 공고를 만들어 보여주지 않는다. 연결 절차는 [docs/DEPLOY.md](docs/DEPLOY.md) 참고.
- 청약홈 API(공공데이터포털 15098547) 실측 스펙 기준 정규화 어댑터와 Supabase 프록시 함수 코드 포함.
- Expo 네이티브 앱(플레이스토어·애드몹)은 로드맵대로 후속 버전에서 진행 ([docs/ROADMAP.md](docs/ROADMAP.md)).

## 저장소 구조

```
zoopzoopcall/
  packages/core/        # 도메인 순수함수 (상태판정·D-day·알림계산·정규화) + Vitest 테스트
  apps/web/             # React + Vite PWA (목록/상세/내 알림/안내, 로컬 알림 스케줄러)
  supabase/functions/   # 청약홈 API 프록시 (서비스키 서버 보관)
  docs/                 # SPEC / ROADMAP / TODO / DEPLOY / DEVELOPMENT_LOG + superpowers 페어
```

## 개발

```bash
pnpm install
pnpm test        # packages/core 순수함수 테스트
pnpm typecheck
pnpm build       # 아이콘 생성 + 타입체크 + 웹 빌드
pnpm --filter @zoopzoopcall/web dev
```

## 원칙 (요약)

- 알림 시각 계산은 순수함수 + 테스트. 과거 시각은 예약하지 않는다.
- 시간은 UTC로 저장, 표시는 Asia/Seoul.
- 접수 일정은 정정될 수 있다는 전제로 설계한다.
- "당첨된다/받을 수 있다" 단정 금지 — 항상 청약홈 확인 톤.
- 신청은 우리가 하지 않는다 — 청약홈 공식 딥링크만.
- 서비스키는 서버(Edge Function)에서만. 앱 번들에 넣지 않는다.

데이터 출처: 한국부동산원 청약홈 분양정보 조회 서비스(공공데이터포털). 신청·자격 확인은 [청약홈](https://www.applyhome.co.kr)에서.
