# 앱 우선순위 점수표 — 오늘 어느 앱을 개선할까

마스터가 매일 각 앱을 점수화해 **가장 높은 앱 1개**를 고른다.
단, 위험 점수가 큰 작업은 코드 수정 대신 **제안서**만 만든다.

| 항목 | 점수 |
|---|---|
| CI 실패 | +50 |
| 빌드 실패 | +50 |
| 기존 배포 위험 | +40 |
| blocked_reason 존재 | +30 |
| CHANGELOG Next 존재 | +20 |
| 사용자 가치 큰 작은 개선 | +15 |
| 오래 방치됨 | +10 |
| 변경 범위 작고 테스트 가능 | +10 |
| 신규 앱 초기 세팅 필요 | +10 |
| secret/API/OAuth/DB 관련 | −100 |
| base path 위험 | −80 |
| 대규모 구조 변경 필요 | −70 |
| 테스트 없이 배포 위험 | −60 |

## 규칙
- 최고점 앱 선택. 음수 위험이 큰 작업은 "제안서 PR"로 전환.
- stub 앱(runningcall/pushrun)은 "초기 세팅" 작업이 있을 때만 후보.
- 매일 한 앱만. 나머지 앱은 최소 **주 1회 touch** SLA로 방치 방지.
