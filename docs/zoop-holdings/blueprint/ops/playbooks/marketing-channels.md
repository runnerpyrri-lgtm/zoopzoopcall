# 다채널 홍보 플레이북 — "매일 양껏 만들고, 발행은 사람"

## 대원칙
1. **자동 대량 게시 금지.** 유튜브/인스타/커뮤니티에 봇으로 댓글·글을 뿌리면 스팸정책 위반 →
   계정 밴 → 브랜드 손상. 이건 홍보가 아니라 자해다.
2. 홍보팀은 **초안만** 만든다. 실제 게시는 사장이 검토 후 직접(또는 승인된 공식 API 워크플로).
3. 커뮤니티 답글은 **광고가 아니라 진짜 도움**이어야 한다. 링크는 맥락상 자연스러울 때만.

## 채널별 접근 (안전하고 효과적인 방식)
| 채널 | 자동(만들기) | 사람(발행) | 비고 |
|---|---|---|---|
| 유튜브 쇼츠 | 대본·훅·썸네일 문구 | 촬영/업로드 | 검색+떡상 노림 |
| 인스타/릴스 | 카드뉴스·캡션·해시태그 | 업로드 | 저장·공유 유도 |
| 블로그/SEO | 글 초안·키워드·메타 | 게시 | 장기 검색 유입(복리) |
| 커뮤니티 답글 | 도움형 답글 초안 | 사람이 판단해 게시 | 스팸 절대 금지 |
| 앱스토어(추후) | 설명·스크린샷 카피 | 등록 | 네이티브 단계 |

## 요일 로테이션 (예시 — 앱 3개 기준)
- 월/목: zoopzoopcall  · 화/금: runningcall · 수/토: pushrun · 일: 성과 회고
- 앱이 늘면 로테이션 주기만 늘린다.

## 성장 루프 (매일 → 반영)
초안 생성 → 사장 게시 승인 → 유입·반응 측정(state 기록) → 다음 카피 개선 → 반복.

## 반자동화 로드맵 (나중에)
- 공식 API가 있고 ToS를 지키는 채널만(예: 예약 발행) 승인 후 워크플로로 자동화 검토.
- 그래도 "무엇을 언제 올릴지"의 최종 결정권은 사장.
