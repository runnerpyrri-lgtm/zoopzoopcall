## [Unreleased]

### Reliability

- 공고 번호가 일부만 제공되는 공고에도 접수일을 포함한 안정적인 식별자를 사용하고, 기존 알림·스냅샷 키는 새 식별자로 자동 이전한다.
- 공고 원문 주소는 `http`와 `https`만 허용해 잘못되거나 위험한 주소가 링크로 노출되지 않게 했다.
- 서비스워커 캐시를 v8로 갱신하고, 오프라인 이동·알림 클릭 시 기존 앱 창 재사용을 안정화했다.
- 앱과 루트 버전을 `0.3.2`로 맞추고 앱별 개인정보처리방침 주소를 연결했다.

- 아파트와 열린 문으로 청약 기회를 표현한 패밀리 대표 아이콘을 파비콘·PWA·알림·앱 헤더에 적용했다.
- 기존 설치 캐시와 구분되는 v2 자산 경로를 사용하고, 빌드 아이콘 생성기도 같은 디자인을 재현한다.
- 공고 유형 필터를 700px 미만에서는 2열 동일 폭으로, 700px 이상에서는 4칸과 지역 선택을 한 줄로 정렬해 `취소후재공급`이 단독 행으로 밀리던 문제를 고쳤다.
- 모바일에서 유형·지역·상태는 바로 고르고, 월 일정만 56px 요약 버튼으로 필요할 때 펼치게 해 첫 공고에 더 빨리 도달하도록 했다. 넓은 화면에서는 월 일정을 기본 펼침으로 유지한다.
- 접수 시작 알림을 공고의 KST 09:00 기준으로 명시하고, 가장 가까운 예약 시각에 맞춰 타이머를 재무장해 15초 점검만 쓰던 지연 가능성을 줄였다.
- 공개 원천을 주소까지 대조해 확인한 단지는 `총 N세대 중 이번 모집 N세대`로 표시한다. 확인되지 않은 단지는 임의 추정하지 않는다.
- 공고 카드의 중복 모집 정보를 한 줄로 합치고, 주택 형태와 원시 주택형을 분리해 `아파트 · 84.6013D형`처럼 정확하게 표시한다.
- 단지 전체 세대수와 이번 모집 세대수를 구분해 표시하며, 원본에 전체 세대수가 없으면 임의로 추정하지 않고 공고문 확인을 안내한다.
- 카드 여백과 정보 구조, 상단·상태 문구를 다듬어 모바일과 넓은 화면 모두에서 더 빠르게 읽히도록 했다.
- 첫 공고만 크게 보이던 구성을 없애고, 이번 주 접수 캘린더와 동일한 요약 카드 목록으로 통일했다.
- 카드에 모집 세대, 분양가, 주택·면적, 당첨 발표, 접수 일정을 데이터가 있는 범위에서 표시하고 밝은 봄 테마를 고정했다.

## [0.3.0] - 2026-07-12

### Changed

- 이번 주 접수 캘린더의 날짜를 눌러 그날 접수·마감 공고만 리스트로 볼 수 있게 했다(다시 누르거나 "전체 보기"로 해제). 이전에는 날짜 칸이 클릭되지 않았다.
- 공고 카드를 좌우 여백을 가진 박스형 카드로 바꾸고, 분양가·면적·접수 일정이 "…"로 잘리던 문제를 줄바꿈 허용으로 고쳐 정보가 온전히 보이게 했다.
- 데이터가 없는 값의 안내 문구를 "공고문 확인"으로 통일했다.
- 앱 셸 캐시(zzc-v6)를 올려 설치 사용자에게 새 화면이 갱신되도록 했다. 기존 저장 키(zzc:*)·manifest·`/homebom/` 배포 경로는 유지했다.

- 실공고에서 가장 시급한 공고를 고르는 히어로, 곧 열리는 기회 목록과 공고·내 알림·설정 하단 내비로 모바일 화면을 전면 재디자인했다.
- 다른 로봄 앱, 문의·광고 제휴, 정책·라이선스, 개발자와 실제 앱 버전을 설정에 통합했다.
- `zzc:subs:v1`, `zzc:fired:v1`, `zzc:notice-snapshots:v1`과 기존 알림 스케줄러는 유지하고 새 앱 셸이 갱신되도록 캐시를 올렸다.

## [0.1.5] - 2026-07-11

- 서비스워커 `install` 단계에서 앱 셸(매니페스트·아이콘 3종)을 미리 캐시해 PWA 설치 직후 오프라인 신뢰성을 높였다.
- 캐시 경로는 sw.js 기준 상대경로만 사용해 base path(`/homebom/`)를 하드코딩하지 않았고, 개별 자산 실패가 설치를 막지 않게 했다.
- 캐시 이름을 `zzc-v2`에서 `zzc-v3`로 올려 기존 사용자도 새 캐시로 전환되게 했다.

- 로봄 포트폴리오 개편에 맞춰 서비스명을 줍줍콜에서 청약봄으로 변경했다.
- PWA 이름, 화면 문구, 저장소 링크와 GitHub Pages base path를 `/homebom/`으로 통일했다.
- 기존 `zzc:*` 저장 키와 `@zoopzoopcall/*` 내부 패키지명은 사용자 설정 및 코드 호환을 위해 유지한다.
- 목록에서는 맥락 없는 선행 알림 권한 요청을 숨겨 첫 공고를 더 빨리 보여주고, 상세에서는 모집공고 원문 다음에 알림 설정을 배치했다.
- 알림 권한이 실제로 허용된 경우에만 공고를 구독하며, 권한 배너와 구독 스위치가 같은 상태를 표시하도록 맞췄다.
- 상세 알림 스위치에 공고명을 포함한 접근 가능한 이름과 44px 터치 영역을 적용했다.

## [0.1.4] - 2026-07-10

유지보수·안정화 릴리스. 기능 변화 없이 버그 재발 방지와 장애 내성을 강화했다.

### 변경
- **버전 통일**: 루트·`apps/web`·`packages/core` 의 package.json 버전이 0.1.3 / 0.1.1 / 0.1.0 으로 제각각이던 것을 모두 **0.1.4** 로 통일했다. 앞으로는 세 파일을 같은 버전으로 유지한다.
- **CI 빌드 환경 일치**: CI(`ci.yml`) 빌드 단계에 배포(`deploy-pages.yml`)와 동일한 공개 실공고 엔드포인트(`VITE_NOTICES_URL`)를 명시해, CI가 프로덕션과 같은 번들 구성을 검증하도록 맞췄다. (공개 URL이며 비밀값 아님)
- **서비스워커 캐시 갱신**: 캐시 이름 `zzc-v1` → `zzc-v2`. activate 시 이전 버전 캐시는 기존 로직대로 자동 삭제된다(네트워크 우선 동작 동일).

### 추가
- **스케줄러 회귀 테스트 6건** (`apps/web/src/notify/__tests__/scheduler.regression.test.ts`):
  - 알림 권한이 `granted` 가 아닐 때 `check` 가 아무것도 fired 로 기록하지 않는지(v0.1.0 영구 억제 버그 재발 방지) + granted 대조군.
  - `collectDueAlerts` 의 6시간 유예창 경계(5시간 59분 경과 = 수집, 6시간 1분 경과 = 폐기)와 이미 fired 된 알림 제외.
  - `delivering` 인플라이트 가드: 같은 알림을 동시에 두 번 전달 시도해도 한 번만 표시·기록.
- **자가개선 워크플로** (`.github/workflows/daily-self-improve.yml`): 하루 2회(KST 06:00/18:00) Claude가 작은 개선 1개를 draft PR 로 올린다. `ANTHROPIC_API_KEY` 시크릿이 없으면 한국어 안내와 함께 즉시 실패한다(등록은 사람 작업). merge 는 항상 사람이 한다.

### 안정화
- **Edge Function 장애 내성** (`supabase/functions/notices/index.ts` — 배포는 사람이 수동으로):
  - stale-if-error: 청약홈 업스트림 장애 시 502 대신 마지막 성공 응답을 `X-Data-Stale: 1` 헤더와 함께 서빙.
  - 업스트림 호출에 AbortController 8초 타임아웃 추가.
  - IP당 분당 30회 인메모리 rate limiter(인스턴스별 best-effort; 플랫폼 차원의 정식 제한은 사람 작업).
  - 공고 정규화 로직·응답 형태는 무변경.
- **웹 무한 로딩 방지** (`apps/web/src/hooks/useNotices.ts`): 실공고 요청에 10초 AbortController 타임아웃을 걸어, 응답이 멈추면 로딩이 아니라 에러 상태로 전환된다.

### 정리
- **회장님 승인 하에** 낡은 스냅샷 디렉터리 삭제: `zoop-holdings/`(구 지주회사 저장소 복사본, VERSION 0.2.0 — 실제 저장소는 runnerpyrri-lgtm/Zoop-holdings), `docs/zoop-holdings/`(구 청사진 스냅샷). 앱 코드가 참조하지 않음을 grep 으로 확인했다.

## [0.1.0] - 2026-07-08

### 추가

- `packages/core`: Notice 도메인 타입, 상태 판정(`getNoticeStatus`/`isClosingSoon`), KST D-day·표기·남은시간(`ddayKst`/`formatKstDateTime`/`formatRemaining`/`formatManwon`), 알림 계산(`buildNoticeAlerts`, 시작 [1일·3시간·정각] / 마감 [1일·3시간·1시간] 프리셋, 과거 미예약), 청약홈 API(15098547) 실측 스펙 정규화(`normalizeRemndrItems`). Vitest 테스트 31건 + 골든마스터.
- `apps/web`: React+Vite PWA. 공고 목록(유형·지역·접수중 필터, 접수중/예정/마감 그룹), 상세(1초 카운트다운, 알림 프리셋 토글, 청약홈 딥링크), 내 알림(예약 목록·5초 테스트 알림·권한 안내), 안내 화면. 로컬 알림 스케줄러(15초 폴링+화면복귀 체크, 발송 이력 중복 방지). 서비스워커(오프라인 셸 캐시, 알림 클릭 → 청약홈). 도장 스탬프 D-day·공고문 종이 디자인, 라이트/다크, 반응형, 접근성(포커스·모션 최소화).
