- PDF.js 텍스트 조각을 좌표 기반 행으로 복원해 공고문 라벨 파싱이 작동하도록 고쳤다. 리디렉션·용량·페이지 제한과 스트리밍 검사를 추가하고 일시 요청 실패를 `미제공`으로 오판하지 않는다.
- 다른 로봄 앱의 캐시를 삭제하던 서비스워커 활성화 로직을 `zzc-` 캐시 범위로 제한했다. 상세 딥링크의 로딩·오류 상태, 달력의 공고 접근성 이름, 날짜 아젠다에서 알림 영역으로 이동하는 동작도 바로잡았다.
- 430px 이하 달력 셀을 66px에서 48px로 줄여 첫 공고가 고정 하단 메뉴 아래로 밀리는 모바일 세로 마찰을 줄였다.
- 서비스워커와 설정의 캐시 표시를 `zzc-v24`로 맞추고 기존 공고 ID, 구독 키, localStorage 키와 `/homebom/` 경로를 유지했다.

## 0.12.0

- 청약홈 API 원시 행, 정규화된 공고, 브라우저 응답과 LKG에 같은 런타임 데이터 계약을 적용해 잘못된 한 행이 전체 화면을 깨지 않게 했다.
- 서비스 역할 전용 영구 공개 스냅샷, 공식 공고 HTML·PDF 캐시와 필드 충돌 기록 테이블을 추가했다. 업스트림 장애 시 검증 성공 시각을 보존한 활성 공고 스냅샷만 stale 응답으로 제공한다.
- 공식 문서 동기화는 Vault 토큰과 서비스 전용 해시를 사용하며, Edge 자원 한도 안에서 15분마다 미처리 공고 한 건씩 순환한다.
- 일반공급 주택형 cold cache를 공개 응답 전에 제한된 동시성으로 채우고, 캐시 행의 실제 `fetched_at`을 모델 확인 시각으로 사용한다.
- 날짜만 제공된 일정을 확정 09:00·17:30으로 표시하지 않는다. 공식 문서에서 시각이 확인된 일정만 초 단위 카운트다운과 시각 기반 알림을 제공한다.
- 비공식 기사와 수동 단지 프로필의 총세대수 보강을 제거했다. 공식 자료에 없는 선택 필드는 대체 문구를 반복하지 않고 카드의 해당 행이나 섹션을 숨긴다.
- 접수 종료 또는 실제 취소 공고를 공개 API, LKG, 달력, 목록과 알림 예약에서 제거하고 `마감·취소` 상태 탭도 삭제했다. 활성 `불법행위 재공급` 유형은 유지한다.
- 실제 신청 딥링크가 아닌 청약홈 루트 버튼은 `청약홈 열기`로 표시하며 공고 원문 링크와 분리했다.
- main push CI와 Pages 배포에 타입 검사, 단위 테스트, Playwright, base path, 서비스워커 문법과 공개 API 계약 검사를 모두 추가했다.
- Playwright fixture 빌드가 Pages 산출물을 덮지 않도록 배포 직전에 운영 API 번들을 다시 생성하고, 테스트 주소가 남으면 배포를 차단한다.
- 서비스워커와 설정의 캐시 표시를 `zzc-v23`으로 맞추고 기존 공고 ID, 구독 키, localStorage 키와 `/homebom/` 경로를 유지했다.
